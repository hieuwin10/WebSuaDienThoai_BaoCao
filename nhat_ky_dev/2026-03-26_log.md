## 🔄 LUỒNG HOẠT ĐỘNG (APPLICATION FLOW)

### 1. Luồng Đăng nhập (Login Flow)
1. Người dùng nhập `username/password` tại `/auth/login`.
2. Backend kiểm tra: `isDeleted`, `lockTime` (nếu sai > 5 lần), sau đó hash so sánh với DB.
3. Nếu đúng: Tạo **JWT Token** chứa `_id` và `role`.
4. Gửi Token về trình duyệt qua **HttpOnly Cookie** tên `TOKEN_C3`.
5. Redirect người dùng về `/profile`.

### 2. Luồng Phân quyền (RBAC Flow)
1. Mỗi user có 1 `role` (ObjectId - ref `role`).
2. Middleware `CheckLogin` giải mã Cookie để lấy thông tin User.
3. Middleware `checkRole('ADMIN', ...)` so sánh role của User với danh sách cho phép.
4. Nếu không khớp: Trả về **403 Forbidden**.

### 3. Luồng Quên mật khẩu (Forgot Password Flow)
1. User nhập email -> Backend tạo `forgotPasswordToken` (32 bytes ngẫu nhiên).
2. Lưu token vào DB cùng thời gian hết hạn (10 phút).
3. Gửi email chứa link: `http://localhost:3000/auth/reset-password/<token>`.
4. User click link -> Nhập mật khẩu mới -> Backend update DB & xoá token.

## 📚 THAM CHIẾU TỪ PROJECT MẪU (THAY_CODE_MAU)

Để đảm bảo tính đồng bộ, dưới đây là các điểm quan trọng từ API của thầy mà chúng ta đã kế thừa và tối ưu:

### 1. Endpoint & Cookie
- **Cookie Name:** Thầy dùng `TOKEN_NNPTUD_C3`. Tôi dùng `TOKEN_C3` (ngắn gọn hơn).
- **Cấu trúc Auth:** Tương đồng 100% về luồng (Login, Register, Logout, Me, Forgot/Reset Password).
- **Transaction:** Trong logic `register` của thầy có thêm phần tạo `Cart` đồng thời với `User` sử dụng MongoDB Session/Transaction. Đây là phần chúng ta sẽ phát triển tiếp khi làm đến module Giỏ hàng.

### 2. Các điểm lưu ý kỹ thuật (Technical Notes)
- **Hashing:** Thầy dùng `bcrypt` trong controller/validator. Tôi đã chuyển hẳn vào **Mongoose Middleware (hooks)** để luôn đảm bảo mật khẩu được hash dù tạo từ bất kỳ đâu (API, Seed, hay Admin panel).
- **Validation:** Thầy sử dụng `RegisterValidator` và `validatedResult` tương tự như cấu trúc tôi đã triển khai, đảm bảo dữ liệu đầu vào sạch.
- **Roles:** Trong code mẫu, thầy gán cứng ID role (`69b6231b3de61addb401ea26`). Hệ thống của tôi linh hoạt hơn bằng cách **truy vấn động theo tên Role (ADMIN/USER)** trong database.
